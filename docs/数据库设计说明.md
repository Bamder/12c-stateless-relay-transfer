# 数据库设计说明

**12C Stateless Relay Transfer — 数据库与运维审计设计**

本文档为项目报告中的数据库设计片段，重点说明新增的运维审计与存储健康记录功能。设计目标是满足关系数据库与 SQL 要求，同时保持项目匿名、无状态中继传输的核心特性。

## 1. 数据库职责

系统采用 Registry / Relay 分库。Registry 负责中继节点、路由、密钥元数据和全局审计；Relay 负责本地块索引、访问结果和清理记录。数据库只保存运行所需的状态和统计，不保存文件内容。

## 2. 功能扩展说明

新增功能为运维审计与存储健康记录：记录心跳、容量变化、上传预留、下载解析、Replica 放弃、本地 PUT/GET 和清理结果。该功能方便演示、排错和后期统计，不引入用户系统或文件管理系统。

## 3. 分库结构

| 数据库 | 表数量 | 主要职责 |
| --- | ---: | --- |
| Registry SQLite | 12 | 中继入池、心跳状态、token 路由、密钥元数据、全局运维审计 |
| Relay SQLite | 3 | 本地块索引、块访问记录、过期/孤儿文件清理记录 |
| **合计** | **15** | 满足优秀档位「数据库不少于 15 张表」的要求 |

## 4. 表结构清单

| 数据库 | 表名 | 主键 | 用途 |
| --- | --- | --- | --- |
| Registry | `registry_allowlist` | `relay_id` | 中继白名单，控制允许接入的 Relay |
| Registry | `relay_registration_requests` | `install_id` | 中继入池申请与审批状态 |
| Registry | `relay_states` | `relay_id` | 当前心跳状态和容量 |
| Registry | `token_relay_placements` | `token`, `relay_id` | token 到 Relay 的主副本路由 |
| Registry | `relay_registry_keys` | `relay_id`, `key_id` | registryApiKey 哈希和剩余次数 |
| Registry | `relay_block_auth_keys` | `relay_id`, `key_id` | 覆盖授权 HMAC 派生密钥元数据 |
| Registry | `relay_heartbeat_events` | `event_id` | 心跳历史和容量变化 |
| Registry | `token_reservation_batches` | `batch_id` | 上传预留请求批次 |
| Registry | `token_reservation_items` | `batch_id`, `token_hash` | 批次内 token 和 blockHash 摘要 |
| Registry | `token_resolution_events` | `event_id` | 下载解析请求统计 |
| Registry | `replica_abandon_events` | `event_id` | Replica 补传失败后的放弃记录 |
| Registry | `registry_admin_events` | `event_id` | 管理操作审计记录 |
| Relay | `blocks` | `token` | 本地 token 到磁盘块路径映射 |
| Relay | `block_access_events` | `event_id` | PUT/GET/覆盖/缺失访问结果 |
| Relay | `block_sweep_runs` | `run_id` | 过期块和孤儿文件清理结果 |

## 5. 核心流程

| 流程 | 数据库行为 |
| --- | --- |
| 上传 | 客户端先向 Registry 预留 token，写入 `token_relay_placements`，同时记录 reservation batch/items；Relay PUT 成功后记录 `block_access_events`。 |
| 下载 | 客户端向 Registry 解析 token，写入 `token_resolution_events`；Relay GET 成功或缺失都会记录访问结果。 |
| 心跳 | Relay 定期上报容量，更新 `relay_states`，并追加 `relay_heartbeat_events` 用于查看历史变化。 |
| 清理 | Relay 清理过期 DB 行和孤儿文件后，写入 `block_sweep_runs`，便于确认清理是否执行。 |

## 6. 安全与隐私

- 不新增用户表，不保存文件名、文件内容或业务身份信息。
- 审计表中的 token 使用 SHA-256 哈希和短前缀（`token_prefix` 为前 8 字符），便于排错但避免长期保存完整凭证。
- 密钥类字段只保存哈希或元数据，管理端数据库浏览继续做脱敏展示。
- 新增功能服务于运维、演示和排错，不改变匿名传输和无状态中继设计。
